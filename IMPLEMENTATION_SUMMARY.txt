          they got "FK constraint violation" error

Solution: Changed 1-booking-per-day limit to allow multiple bookings with
          1-hour minimum gap between same restaurant bookings

Changes:  - Added canBookAfterMinimumGap() function
          - Updated booking validation logic

          ✅ Monday 2:00 PM - OK (4-hour gap)

NO MORE FK ERRORS!
